Arthas -- Java 应用在线诊断神器

发表于 2019-03-23

Arthas 是什么

Arthas 是Alibaba开源的Java诊断工具，深受开发者喜爱。

Arthas 是基于 Greys 进行二次开发的全新在线诊断工具，支持JDK 6+，支持Linux/Mac/Winodws, 利用 Java6 的 Instrumentation 特性，动态增强你所指定的类，获取你想要到的信息, 采用命令行交互模式，同时提供丰富的 Tab 自动补全功能，进一步方便进行问题的定位和诊断。

Arthas 能做哪些事

当你遇到以下类似问题而束手无策时，Arthas可以帮助你解决：

  • 这个类从哪个 jar 包加载的？为什么会报各种类相关的 Exception？
  • 我改的代码为什么没有执行到？难道是我没 commit？分支搞错了？
  • 遇到问题无法在线上 debug，难道只能通过加日志再重新发布吗？
  • 线上遇到某个用户的数据处理有问题，但线上同样无法 debug，线下无法重现！
  • 是否有一个全局视角来查看系统的运行状况？
  • 有什么办法可以监控到JVM的实时运行状态？

GitBook初步认识

发表于 2018-11-23

GitBook是一种使用Markdown语法编写的的电子书，并基于Git进行版本管理。

为什么要选择GitBook

Markdown语法简单，格式简洁，编写快速。

Git是目前最优秀的版本控制工具之一，使用Git和Git仓库来管理文档，即确保了文档存储的持久性，也可随时查看历史版本。

GitBook的主要用户群是程序员。MarkDown和Git已经是程序员的必备技能了，因此对于程序员而已，使用GitBook来写电子书或文档是很好的选择

GitBook的使用场景

GitBook只是一种编辑电子书的技术，可以根据个人需求灵活使用。下面分享一下我看到的GitBook的使用场景

生成GitHub项目的文档

以前GitHub项目的文档也是使用Markdown写的，比如README.MD。
不过这种文档和代码混在一起，也比较零散，体验不大好。

现在已经有不少项目开始使用GitBook结合github-pages生成项目的文档。

大致的流程是：使用GitBook编辑文档并生成静态页面，然后推送到GitHub项目的gh-pages分支，然后通过 https://username.github.io/projectname来访问Gitbook电子书。

whistle -- 跨平台抓包调试代理工具

发表于 2018-11-15

应用场景

在日常开发工作中，经常需要抓取Http包来调试前后端的数据传递，此外，为了便于调试，有时也会需要对前端发起的对远程服务器的Http请求转发到本机等。这时就要用到Http代理调试工具来实现以上场景。

为什么要用whistle

说到Http代理工具，windows系统中最常用的应该是fiddler。fiddler是一款功能十分强大的Http代理调试工具，但长期使用会很占内存，经常卡死，这时就只能使用重启大法了，而且fiddler的编写rule的脚本也比较复杂，编写难度高。

在macOS系统中，比较常用的Http代理工具是Charles,这款应用颜值很高，使用也挺简单的，但是需要购买(当然也可以下载网络上的破解版)

那么，对于以上两款常用的Http代理工具，whistle有什么优势呢

  • 跨平台。whistle基于node编写，通过浏览器访问，具有跨平台的特点。
  • 占内存少。whistle只是启动一个后端服务，占内存较少，不用担心卡死问题。
  • rule脚本简单。whistle的rule脚本是基于node语法的，相对比较简单。
  • 免费，代码开源。whistle的代码发布在github上，可以参考学习。

yarn简单入门

发表于 2018-11-14

yarn是什么

yarn是由Facebook、Google、Exponent 和 Tilde 联合推出了一个新的 JS 包管理工具。

为什么要使用yarn

说到JS包管理工具，大家的第一反应自然是npm，npm是官方提供的包管理工具。
有了npm，为什么还要转用yarn呢，yarn解决了什么样的问题？

npm的缺点（npm5.0之前的版本）

在yarn诞生之前，在使用npm管理包的过程中发现存在一些问题,主要有以下两点

  • 每次npm install耗时长
  • 不同机器或者环境下，npm install 下载的版本不一致，导致偶尔会出现一些奇怪的bug

yarn的改进方案

提高包的下载速度

  • 并行下载。并行下载以最大化资源利用率，因此安装速度更快。
  • 缓存。缓存了每个下载过的包，所以再次使用时无需重复下载。缓存过的包，离线模式也可下载

保证依赖包版本的确定性

  • 使用yarn.lock文件记录依赖包的确定版本，yarn install根据yarn.lock的依赖包版本来下载，保证依赖包的版本在每个环境都是一致的
