Profile 否

发表于 2019-10-08

文档

官方文档

If a @Configuration class is marked with @Profile, all of the @Bean methods and @Import annotations associated with that class are bypassed unless one or more of the specified profiles are active. If a @Component or @Configuration class is marked with @Profile({“p1”, “p2”}), that class is not registered or processed unless profiles ‘p1’ or ‘p2’ have been activated. If a given profile is prefixed with the NOT operator (!), the annotated element is registered only if the profile is not active. For example, given @Profile({“p1”, “!p2”}), registration will occur if profile ‘p1’ is active or if profile ‘p2’ is not active.

常见使用场景

消费者 bean 在本地开发环境不应该被初始化，避免消费环境上的消息。

Spring 工具类

发表于 2019-07-26

Spring 是目前使用最广泛的 Java 框架，基本所有 Java EE 应用都在使用 Spring。

而 Spring 本身带有很多有用的工具类，灵活得运用这些工具类可以简化日常的代码开发。

为什么要使用 Spring 的工具类

Spring 工具类对于其他工具类的优势主要有:

  • 基本所有 Java 应用都在使用 Spring 框架，使用 Spring 工具类无需再引入额外依赖
  • Spring 的代码设计和稳定性都比较好，工具的性能和稳定性有比较好的保障

基于以上优势，如果需要用到某个工具方法，而 Spring 工具类中存在这个方法，建议尽量使用 Spring 工具类的该方法。

Arthas -- Java 应用在线诊断神器

发表于 2019-03-23

Arthas 是什么

Arthas 是Alibaba开源的Java诊断工具，深受开发者喜爱。

Arthas 是基于 Greys 进行二次开发的全新在线诊断工具，支持JDK 6+，支持Linux/Mac/Winodws, 利用 Java6 的 Instrumentation 特性，动态增强你所指定的类，获取你想要到的信息, 采用命令行交互模式，同时提供丰富的 Tab 自动补全功能，进一步方便进行问题的定位和诊断。

Arthas 能做哪些事

当你遇到以下类似问题而束手无策时，Arthas可以帮助你解决：

  • 这个类从哪个 jar 包加载的？为什么会报各种类相关的 Exception？
  • 我改的代码为什么没有执行到？难道是我没 commit？分支搞错了？
  • 遇到问题无法在线上 debug，难道只能通过加日志再重新发布吗？
  • 线上遇到某个用户的数据处理有问题，但线上同样无法 debug，线下无法重现！
  • 是否有一个全局视角来查看系统的运行状况？
  • 有什么办法可以监控到JVM的实时运行状态？

GitBook初步认识

发表于 2018-11-23

GitBook是一种使用Markdown语法编写的的电子书，并基于Git进行版本管理。

为什么要选择GitBook

Markdown语法简单，格式简洁，编写快速。

Git是目前最优秀的版本控制工具之一，使用Git和Git仓库来管理文档，即确保了文档存储的持久性，也可随时查看历史版本。

GitBook的主要用户群是程序员。MarkDown和Git已经是程序员的必备技能了，因此对于程序员而已，使用GitBook来写电子书或文档是很好的选择

GitBook的使用场景

GitBook只是一种编辑电子书的技术，可以根据个人需求灵活使用。下面分享一下我看到的GitBook的使用场景

生成GitHub项目的文档

以前GitHub项目的文档也是使用Markdown写的，比如README.MD。
不过这种文档和代码混在一起，也比较零散，体验不大好。

现在已经有不少项目开始使用GitBook结合github-pages生成项目的文档。

大致的流程是：使用GitBook编辑文档并生成静态页面，然后推送到GitHub项目的gh-pages分支，然后通过 https://username.github.io/projectname来访问Gitbook电子书。
