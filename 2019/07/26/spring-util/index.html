Spring 工具类

发表于 2019-07-26

Spring 是目前使用最广泛的 Java 框架，基本所有 Java EE 应用都在使用 Spring。

而 Spring 本身带有很多有用的工具类，灵活得运用这些工具类可以简化日常的代码开发。

为什么要使用 Spring 的工具类

Spring 工具类对于其他工具类的优势主要有:

  • 基本所有 Java 应用都在使用 Spring 框架，使用 Spring 工具类无需再引入额外依赖
  • Spring 的代码设计和稳定性都比较好，工具的性能和稳定性有比较好的保障

基于以上优势，如果需要用到某个工具方法，而 Spring 工具类中存在这个方法，建议尽量使用 Spring 工具类的该方法。

HTTP 请求 – RestTemplate

Java 后端最常见的 Http 客户端工具应该是 apache 的 HttpClient。HttpClient 虽然功能强大，但是开发代码复杂，得用其它工具转换响应数据，还得操心资源回收等。使用底层的 JDK HttpURLConnection 更是繁琐。引入使用 Feign, Retrofit 工具又有点大材小用的感觉。

RestTemplate 是基于底层 HTTP 客户端(如 JDK HttpURLConnection, Apache HttpComponents)封装的一套模板方法API，是 Spring 提供的用于访问 Rest 服务的客户端，RestTemplate 提供了多种便捷访问远程 Http 服务的方法, 能够大大提高客户端的编写效率。

反射工具类 – ReflectionUtils

JDK 反射相关的 API 虽然功能强大，但代码写起来比较冗长，这时可以借助 Spring 的 ReflectionUtils 工具类来简化一些常用的反射代码。

下面展示一下使用 ReflectionUtils 的方法获取私有 Field 的值:

public class ReflectionUtilsTest {

    @Test
    public void testGetPrivateFiled() {
        User user = new User();
        Field field = ReflectionUtils.findField(User.class, "id");
        ReflectionUtils.makeAccessible(field);
        Assert.notNull(ReflectionUtils.getField(field, user), "get privated field error");
    }
}

class User {
    private String id;

    public User() {
        id = new JdkIdGenerator().generateId().toString();
    }
}

ReflectionUtils 提供了不少反射相关的工具方法，需要使用反射建议先找找 ReflectionUtils 里面的方法是否可以使用
