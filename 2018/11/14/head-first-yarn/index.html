yarn简单入门

发表于 2018-11-14

yarn是什么

yarn是由Facebook、Google、Exponent 和 Tilde 联合推出了一个新的 JS 包管理工具。

为什么要使用yarn

说到JS包管理工具，大家的第一反应自然是npm，npm是官方提供的包管理工具。
有了npm，为什么还要转用yarn呢，yarn解决了什么样的问题？

npm的缺点（npm5.0之前的版本）

在yarn诞生之前，在使用npm管理包的过程中发现存在一些问题,主要有以下两点

  • 每次npm install耗时长
  • 不同机器或者环境下，npm install 下载的版本不一致，导致偶尔会出现一些奇怪的bug

yarn的改进方案

提高包的下载速度

  • 并行下载。并行下载以最大化资源利用率，因此安装速度更快。
  • 缓存。缓存了每个下载过的包，所以再次使用时无需重复下载。缓存过的包，离线模式也可下载

保证依赖包版本的确定性

  • 使用yarn.lock文件记录依赖包的确定版本，yarn install根据yarn.lock的依赖包版本来下载，保证依赖包的版本在每个环境都是一致的

yarn和npm的使用方式很像，主要命令只是名称不太一样，流程基本一致。

安装依赖

命令格式: yarn install
等同于: npm install
安装时，如果yarn.lock文件中有相应依赖的信息，则按照yarn.lock中的信息进行下载
如果yarn.lock没有相应信息，则根据package.json中的版本根据相应规则下载并记录到yarn.lock中。

yarn对社区的作用

yarn的出现在一定程度上促进了npm的改进，npm从5.0版本开始，使用package-lock.json来保证依赖包的确定性，也有使用缓存改进install的性能，但目前，yarn的速度依然目前优于npm。

是否要转yarn

yarn对于npm的优势主要在于install的速度，如果追求更快的install速度，可以考虑使用yarn，而且个人觉得yarn的提示信息也比较清晰些。
